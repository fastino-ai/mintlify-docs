---
title: "OpenCode"
---

Steps to integrate OpenCode with Pioneer:

1. Download and [set up](https://opencode.ai/download) the OpenCode CLI.
2. Make your Pioneer API key available in your shell:

```bash
export PIONEER_API_KEY="<Your Pioneer API key>"
```

<Note>
  For plain `opencode` to work in future terminals, keep `PIONEER_API_KEY` in your shell profile or another environment manager. OpenCode will read it from the persisted config below.
</Note>

3. Run this one-time setup command. It fetches the live `GET /v1/models` catalog, merges Pioneer into your global OpenCode config at `~/.config/opencode/opencode.json`, and sets `pioneer/auto` as the default model:

```bash
: "${PIONEER_API_KEY:?Set PIONEER_API_KEY first}"

CONFIG_DIR="${XDG_CONFIG_HOME:-$HOME/.config}/opencode"
CONFIG_FILE="$CONFIG_DIR/opencode.json"
TMP_FILE="$(mktemp)"

mkdir -p "$CONFIG_DIR"
[ -f "$CONFIG_FILE" ] || printf '{}\n' > "$CONFIG_FILE"
cp "$CONFIG_FILE" "$CONFIG_FILE.bak.$(date +%Y%m%d%H%M%S)"

curl -fsS "https://api.pioneer.ai/v1/models" \
  -H "Authorization: Bearer $PIONEER_API_KEY" \
  | jq --slurpfile current "$CONFIG_FILE" '
    def model_name:
      (.display_name // .id)
      | sub(" (?<major>[0-9]+) (?<minor>[0-9]+)(?= |$)"; " \(.major).\(.minor)")
      | gsub("\\bGpt\\b"; "GPT")
      | gsub("\\bOss\\b"; "OSS");

    ($current[0] // {}) + {
      "$schema": "https://opencode.ai/config.json",
      model: "pioneer/auto",
      provider: (($current[0].provider // {}) + {
        pioneer: {
          npm: "@ai-sdk/openai-compatible",
          name: "Pioneer",
          env: ["PIONEER_API_KEY"],
          options: {
            baseURL: "https://api.pioneer.ai/v1",
            apiKey: "{env:PIONEER_API_KEY}"
          },
          models: (
            {"auto": {"id": "pioneer/auto", "name": "Pioneer Auto"}}
            + (
              [.data[]
                | select(.deprecated != true)
                | select(.id != null)
                # Hide Claude Code discovery aliases so OpenCode does not show every model twice.
                | select(.id | startswith("anthropic/pioneer/") | not)
                # Add the router once as the pioneer/auto model below, even if /v1/models includes it.
                | select(.id != "pioneer/auto" and .id != "auto")
                | {key: .id, value: {name: model_name}}]
              | from_entries
            )
          )
        }
      })
    }' > "$TMP_FILE"

mv "$TMP_FILE" "$CONFIG_FILE"
```

4. Start OpenCode normally:

```bash
opencode
```

5. To switch between models, use the `/models` command and select a model. Choose `pioneer/auto` to use Pioneer's Code Router, or choose any specific canonical model returned by `GET /v1/models`.

<Note>
  The model list is pulled when you run the setup command. Re-run the setup command when you want OpenCode to pick up newly added Pioneer models. The command filters out `anthropic/pioneer/*` discovery aliases because they are only needed for Claude Code and would duplicate models in OpenCode.
</Note>
