---
title: "OpenClaw"
---

Pioneer exposes an OpenAI-compatible inference endpoint, so OpenClaw can use Pioneer as a custom provider today. This guide covers the custom-provider setup until Pioneer is available as an official OpenClaw provider.

<Warning>
  OpenClaw is a third-party local agent tool. Keep the gateway bound to loopback unless you have intentionally hardened remote access, channel allowlists, and tool permissions.
</Warning>

### Prerequisites

- OpenClaw installed:

```shellscript
npm install -g openclaw@latest
```

- A Pioneer API key from the Pioneer dashboard.
- `jq` installed for converting the live Pioneer model catalog into OpenClaw config.
- Check to see if OpenClaw is available:

```shellscript
openclaw --version
```

### Setup steps

1. Save Pioneer auth in OpenClaw
   - Use OpenClaw's model auth store for the Pioneer API key. This is more reliable than relying on shell or `launchctl` environment propagation, because the OpenClaw gateway runs as a service.
     ```shellscript
     openclaw models auth paste-api-key --provider pioneer
     ```
   - Paste your Pioneer API key when prompted.
     <Note>
       Do not paste real API keys into docs, screenshots, shared shell history, or issue trackers. Rotate the key if it has been exposed.
     </Note>
2. Discover and register Pioneer models
   - Pioneer's supported inference models can change over time. Use the OpenAI-compatible `GET /v1/models` catalog as the source of truth and convert it directly into OpenClaw model config.
   - Set `PIONEER_API_KEY` for this one-time catalog fetch. OpenClaw still uses the auth profile from the previous step when it runs the gateway.
     ```shellscript
     export PIONEER_API_KEY='<YOUR_PIONEER_API_KEY>'
     : "${PIONEER_API_KEY:?Set PIONEER_API_KEY first}"

     CONFIG_FILE="$(openclaw config file)"
     case "$CONFIG_FILE" in
       "~/"*) CONFIG_FILE="$HOME/${CONFIG_FILE#"~/"}" ;;
     esac
     [ -f "$CONFIG_FILE" ] || printf '{}\n' > "$CONFIG_FILE"

     curl -fsS "https://api.pioneer.ai/v1/models" \
       -H "Authorization: Bearer $PIONEER_API_KEY" \
       | jq --slurpfile current "$CONFIG_FILE" -c '
         def model_name:
           (.display_name // .name // .id)
           | split("/") | last
           | gsub("_"; " ")
           | gsub("-"; " ")
           | sub(" (?<major>[0-9]+) (?<minor>[0-9]+)(?= |$)"; " \(.major).\(.minor)")
           | gsub("\\bGpt\\b"; "GPT")
           | gsub("\\bOss\\b"; "OSS")
           | gsub("\\bAi\\b"; "AI")
           | gsub("(?<n>[0-9]+(\\.[0-9]+)?)b\\b"; "\(.n)B");

         def openclaw_model_id($id):
           if $id | startswith("pioneer/") then $id else "pioneer/" + $id end;

         (
           [
             {
               id: "pioneer/auto",
               name: "Pioneer Auto",
               input: ["text"],
               contextWindow: 1000000,
               maxTokens: 16000
             }
           ] + (
             [
               .data[]
               | select(.deprecated != true)
               | select(.id != null)
               | select(.id | startswith("anthropic/pioneer/") | not)
               | select(.id != "pioneer/auto" and .id != "auto")
               | {
                   id: .id,
                   name: model_name,
                   input: (["text"] + (if .capabilities.image_input.supported == true then ["image"] else [] end)),
                   contextWindow: (.max_input_tokens // .context_window // 128000),
                   maxTokens: (.max_tokens // 16000)
                 }
             ]
             | unique_by(.id)
             | sort_by(.id)
           )
         ) as $pioneer_models
         | ($current[0].agents.defaults.models // {}) as $current_agent_models
         | {
             models: {
               providers: {
                 pioneer: {
                   baseUrl: "https://api.pioneer.ai/v1",
                   api: "openai-completions",
                   models: $pioneer_models
                 }
               }
             },
             agents: {
               defaults: {
                 model: {primary: "pioneer/auto"},
                 models: (
                   ($current_agent_models | with_entries(select(.key | startswith("pioneer/") | not)))
                   + (
                     $pioneer_models
                     | map({key: openclaw_model_id(.id), value: {alias: .name}})
                     | from_entries
                   )
                 )
               }
             }
           }' \
       | openclaw config patch \
           --stdin \
           --replace-path models.providers.pioneer.models \
           --replace-path agents.defaults.models
     ```
   - The command manually adds `pioneer/auto` for Pioneer Auto, then reads `.data[]`, deduplicates by `id`, filters `anthropic/pioneer/*` Claude Code discovery aliases so OpenClaw does not show duplicate models, and exposes every Pioneer model under `agents.defaults.models` for the model picker and `openclaw models status`.
   - To refresh the catalog later, re-run the same command. It replaces the Pioneer provider models and Pioneer agent allowlist while preserving non-Pioneer agent model entries.
3. Check the discovered models
   - Confirm OpenClaw sees the generated Pioneer provider catalog:
     ```shellscript
     openclaw models list --provider pioneer --plain
     ```
   - Confirm the default agent allowlist also contains the Pioneer models. `models status --plain` prints only the default model, so use JSON output to check the full allowlist:
     ```shellscript
     openclaw models status --json \
       | jq '{defaultModel, allowed_count: (.allowed | length), first_allowed: .allowed[0:5]}'
     ```
   - Expected result: `defaultModel` is `pioneer/auto`, and `allowed_count` is greater than `1`.
   - The setup command sets Pioneer Auto as the default model. If you need to set it again manually, run:
     ```shellscript
     openclaw models set pioneer/auto
     ```
   - If you choose to use environment variables instead of `openclaw models auth paste-api-key`, configure the header as a structured env reference:
     ```shellscript
     openclaw config set models.providers.pioneer.headers.X-API-Key \
         --ref-provider default \
         --ref-source env \
         --ref-id PIONEER_API_KEY
     ```
   - The gateway service must be able to resolve that variable. In practice, the auth-profile command above is the simpler path.
4. Safely install or restart the gateway
   - For a local setup, keep the gateway on loopback and use the LaunchAgent service. Do not run `openclaw gateway run` during normal setup; that starts a second foreground gateway and will fail if the service already owns the port.
     ```shellscript
     openclaw config set gateway.mode local

     if openclaw gateway status >/tmp/openclaw-gateway-status.txt 2>&1; then
       openclaw gateway restart
     else
       openclaw gateway install
       openclaw gateway restart
     fi

     openclaw gateway status
     ```
   - If OpenClaw reports that the service config is out of date or non-standard, repair the LaunchAgent and restart it:
     ```shellscript
     openclaw doctor --repair
     openclaw gateway restart
     openclaw gateway status
     ```
   - If OpenClaw shows a security disclaimer during setup, read it and continue only if this is a trusted personal setup. Do not expose a tool-enabled gateway to untrusted users.
5. Verify Pioneer through the running gateway
   - Use a live probe to confirm Pioneer auth and model routing actually work. OpenClaw may probe only the default/effective target even when the agent allowlist contains many Pioneer models; that is fine as long as the configured model count is greater than `1` and `pioneer/auto` probes successfully:
     ```shellscript
     openclaw models status --probe --probe-provider pioneer
     ```
   - Expected result:
     ```text
     Configured models (...): pioneer/auto, pioneer/...
     Auth probes
     pioneer/auto | pioneer:manual (api_key) | ok
     ```
6. Open the OpenClaw Web UI
   - OpenClaw's local dashboard is the easiest way to verify the model picker and send a first message through the gateway:
     ```shellscript
     openclaw dashboard
     ```
   - If the gateway is already running as a service, do not start a second foreground gateway with `openclaw gateway run`. Use the dashboard command or restart the service instead:
     ```shellscript
     openclaw gateway restart
     openclaw dashboard
     ```
   - The dashboard normally opens a loopback URL such as `http://127.0.0.1:18789/`. Some local installs use device identity instead of a copy-paste gateway token; in that case, follow the browser pairing prompt or run:
     ```shellscript
     openclaw doctor --repair
     ```
7. Run a first agent message from the CLI
   - OpenClaw needs a target session for agent messages. A plain `--message` is not enough.
     ```shellscript
     openclaw agent --agent main --session-key cli-test --message "hello"
     ```
   - To continue the same local session:
     ```shellscript
     openclaw agent --agent main --session-key cli-test --message "summarize the previous answer"
     ```
   - You can list sessions with:
     ```shellscript
     openclaw sessions list
     ```

### Troubleshooting OpenClaw integration

1. `Missing auth - pioneer`
   - OpenClaw does not have a Pioneer auth profile. Run:
     ```shellscript
     openclaw models auth paste-api-key --provider pioneer
     openclaw models status --probe --probe-provider pioneer
     ```
2. `Missing env var "PIONEER_API_KEY"`
   - This usually means the config points at an environment variable that is not visible to the OpenClaw process or gateway service. Prefer the auth-profile setup:
     ```shellscript
     openclaw models auth paste-api-key --provider pioneer
     ```
   - If you intentionally use an env reference, confirm the variable is visible to the process that runs OpenClaw:
     ```shellscript
     test -n "$PIONEER_API_KEY" && echo "set" || echo "not set"
     ```
   - For launchd, `launchctl setenv` does not persist across reboots and may not be enough if OpenClaw uses a generated service environment wrapper.
3. `Gateway service not loaded`
   - Install or restart the LaunchAgent:
     ```shellscript
     openclaw gateway install
     openclaw gateway restart
     openclaw gateway status
     ```
4. `No target session selected`
   - Pass a session target:
     ```shellscript
     openclaw agent --agent main --session-key cli-test --message "hello"
     ```
5. `gateway already running` or `Port 18789 is already in use`
   - This usually means the LaunchAgent service is already bound to the gateway port. That is normal; do not start a second gateway with `openclaw gateway run`.
   - For normal use, restart the service and open the Web UI:
     ```shellscript
     openclaw gateway restart
     openclaw dashboard
     ```
   - If `gateway status` says the service config is out of date, repair and restart:
     ```shellscript
     openclaw doctor --repair
     openclaw gateway restart
     ```
   - For foreground debugging only, stop the service first, then run the gateway in the foreground:
     ```shellscript
     openclaw gateway stop
     openclaw gateway run
     ```
6. Dashboard or probe says `device identity required`
   - Local OpenClaw installs may use device identity instead of a shared token. Repair or re-pair the local CLI/browser identity:
     ```shellscript
     openclaw doctor --repair
     openclaw dashboard
     ```
7. Gateway is running but probe fails
   - Check reachability and logs:
     ```shellscript
     openclaw gateway status
     openclaw gateway probe
     openclaw logs --follow
     ```
